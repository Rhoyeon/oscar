# RAG Setup Skill

## 목적

지식그래프를 LLM이 활용 가능한 RAG 시스템으로 변환한다. 그래프 구조를 보존하면서 의미 검색이 가능한 벡터 인덱스를 구성한다.

## 임베딩 전략

지식그래프의 3가지 단위로 임베딩을 생성한다:

| 단위 | 임베딩 내용 | 용도 |
|-----|-----------|-----|
| **Node** | `{클래스}: {레이블}. {속성 목록}` | 엔티티 검색 |
| **Edge** | `{주어} {관계} {목적어}. {컨텍스트}` | 관계 검색 |
| **Subgraph** | 2-hop 이웃 노드 요약 | 복합 쿼리 |

## 검색 전략

**하이브리드 검색 (기본값)**:
1. BM25 키워드 검색으로 후보 집합 생성 (top-50)
2. 벡터 유사도로 재랭킹 (top-10)
3. 온톨로지 계층 기반 컨텍스트 확장 (부모/자식 클래스 포함)

**컨텍스트 확장 규칙**:
- 검색된 노드의 1-hop 이웃을 자동으로 컨텍스트에 포함
- 온톨로지 상위 클래스 정의를 컨텍스트 앞부분에 추가
- 최대 컨텍스트 크기: 4,000 토큰

## 지원 벡터 DB

| DB | 장점 | 선택 기준 |
|----|-----|---------|
| Chroma | 로컬, 설치 간단 | 개발/소규모 |
| Qdrant | 고성능, 필터링 강력 | 프로덕션 중규모 |
| Pinecone | 완전 관리형 | 프로덕션 대규모 |
| Weaviate | 그래프+벡터 통합 | 지식그래프 특화 |

## 쿼리 인터페이스

- **자연어**: "X와 Y의 관계는?" → 자동으로 벡터 검색 + 컨텍스트 구성
- **SPARQL**: 온톨로지 기반 정형 쿼리
- **Python API**: `oscar.query(text, top_k=10)` 형태
